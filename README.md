## About

I'm a **.NET Backend Engineer** with production experience building Web APIs, MVC applications, and LLM-integrated systems. I'm making a deliberate, structured transition into **AI Engineering** — not by surface-level API calls, but by building the systems underneath: RAG pipelines, autonomous agents, structured outputs, and orchestration logic.

My approach: understand things at the level of first principles, ship real systems, and document everything so knowledge compounds over time.

## Engineering Philosophy

> *"Don't use something you don't understand. Understand it — then use it better than anyone."*

The gap between a good engineer and a great one isn't the number of frameworks they know. It's the depth at which they understand the ones that matter. I read source code when things break. I build before I claim to know something. I document everything — because thinking clearly enough to write it down is how you actually know you understand it.
