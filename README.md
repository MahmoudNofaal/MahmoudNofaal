## Engineering Philosophy

> *"The tools follow the problem — not the other way around."*

I work at the level of architecture and design before I reach for a framework. I read source code when things break. I build before I claim to know something. I document what I understand — because writing clearly enough to explain it is how I know I actually understand it.
